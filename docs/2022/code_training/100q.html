DONE Q4: Search Insert position leetcode   easy

  • Check for empty list at the beginning.
  • Take care of overflow in computing the middle index (python is free of that, though)
  • If found it on the while, don't bother on exiting. Just assign it to left edge.
  • In post-processing:
    • You know for sure that l and r are one at the very side of each other (not crossed)
    • Given that "you have to return the index at which the target should go", then any target smaller or equal than nums[l] will be inserted at index l.
    • Similar for nums[r].
    • If the target is bigger than nums[r], then even after shrinking the search down to the right-most, it was not found, so the target should be appended at the end of the array r+1.
 1: class Solution:
 2:     def searchInsert(self, nums: list[int], target: int) -> int:
 3:         l,r = 0,len(nums)-1
 4: 
 5:         '''if empty list'''
 6:         if r == -1:
 7:             return 0
 8: 
 9:         '''narrow the search down'''
10:         while l+1 < r:
11:             mid = l + (r-l) // 2
12:             if nums[mid] <= target:
13:                 l = mid
14:             else:
15:                 r = mid
16: 
17:         print(f'l:{l}\nr:{r}')
18: 
19:         '''post-processing'''
20:         if target <= nums[l]:
21:             return l
22:         if target <= nums[r]:
23:             return r
24:         else:
25:             return r+1
Runtime \(O(\log(n))\) 64ms 67.56% Memory \(O(1)\) 14.7MB 40.44%

DONE Q3: Merge two sorted Lists leetcode   easy

There are at least two different solutions:

Solution 1

  • Check for empty lists.
  • It could have been solved by copying into a new list, but that uses more memory.
  • The general idea is to:
    1. Pick the list that starts with the smallest element as the leader, and the other as the other list.
    2. Traverse the leader finding a spot where to cut it.
    3. Find a segment from other that will fit in that cut.
    4. Re-route the links.
    5. If one of the lists is exhausted, then we may need to tailor the end of leader, and exit.
    6. If other still has elements, then go back to 1. and repeat.
 1: # Definition for singly-linked list.
 2: # class ListNode:
 3: #     def __init__(self, val=0, next=None):
 4: #         self.val = val
 5: #         self.next = next
 6: 
 7: class Solution:
 8:     def mergeTwoLists(self, list1: Optional[ListNode], \
 9:                       list2: Optional[ListNode]) -> Optional[ListNode]:
10:         '''if one list is not defined, return the other'''
11:         if list1 == None:
12:             return list2
13:         elif list2 == None:
14:             return list1
15: 
16:         '''Rear pointers pointing to:
17:         - lead: the list with the smallest first element
18:         - other: the... other list :)'''
19:         if list1.val < list2.val:
20:             lead_r,other_r = (list1,list2)
21:         else:
22:             lead_r,other_r = (list2,list1)
23: 
24:         '''front pointers that will be ahead of {lead,other}_r'''
25:         lead_f,other_f = lead_r.next,other_r.next;
26: 
27:         '''pointer to return at the end of the function'''
28:         result_head = lead_r
29: 
30:         '''    lr  lf
31:         lead : 1   2   6   8   9
32:         other: 3   4   5   7
33:                or  of
34:         '''
35: 
36:         '''Find the cutting point in 'lead', and the segment from 'other'
37:         to insert it in the cut.'''
38:         while lead_r != None and other_r != None:
39: 
40:             # move the pair [lead_r,lead_f] forward for as long as
41:             # lead_f is smaller than other_r.
42:             while lead_f != None and lead_f.val < other_r.val:
43:                 lead_r = lead_f
44:                 lead_f = lead_f.next
45: 
46:             '''we found the cutting point in lead: right between lead_r
47:             and lead_f. Now we find how long is the segment in 'other'
48:             that will be inserted into lead.'''
49: 
50:             '''if lead_f is None, then lead_r is the last element of 
51:             lead, therefore, we just attach everything from other at the
52:             end of lead; and we are done.'''
53:             if lead_f == None:    
54:                 lead_r.next = other_r
55:                 return result_head
56: 
57:             '''try to expand the segment in other by moving other_f
58:             forward for as long as other_f.val could be before
59:             lead_f.val. Var other_m (middle) keeps track of the node
60:             before other_f'''
61:             other_m = other_r
62:             while other_f != None and other_f.val <= lead_f.val:
63:                 other_m = other_f
64:                 other_f = other_f.next
65: 
66:             '''other_m is now the last element in the segment of other
67:             that goes between lead_r and lead_f. Now we patch the
68:             pointers.
69:             lead : 1   2\            / 6   8   9
70:             other:       \ 3   4   5/  7
71:             '''
72:             lead_r.next = other_r
73:             other_m.next = lead_f
74: 
75:             ''' if other_f is null, then there is nothing else in other,
76:             then, we are done.'''
77:             if other_f == None:
78:                 break
79: 
80:             '''If lead_f is null AND other_f is still something (we did
81:             not brake above), we attach the remainder of other to the
82:             end last element of lead (other_m) before exiting.'''
83:             if lead_f == None:
84:                 other_m.next = other_f
85:                 break
86: 
87:             '''If none of them is null, then there is more work to do.'''
88:             lead_r,other_r = lead_f,other_f
89:             lead_f,other_f = lead_f.next,other_f.next
90:         return result_head
Runtime O(n) 53ms 57.50% Memory O(1) 14MB 31.85%

Solution 2

  • Much simpler, but writes at every element.
 1: # Definition for singly-linked list.
 2: # class ListNode:
 3: #     def __init__(self, val=0, next=None):
 4: #         self.val = val
 5: #         self.next = next
 6: 
 7: class Solution:
 8:     def mergeTwoLists(self, list1: Optional[ListNode], \
 9:                       list2: Optional[ListNode]) -> Optional[ListNode]:
10:         head = tip = ListNode()
11: 
12:         while list1 and list2:
13:             if list1.val < list2.val:
14:                 tip.next = list1
15:                 list1 = list1.next
16:             else:
17:                 tip.next = list2
18:                 list2 = list2.next
19:             tip = tip.next
20: 
21:         if list1:
22:             tip.next = list1
23:         if list2:
24:             tip.next = list2
25: 
26:         return head.next
Runtime O(n) 66ms 28.64% Memory O(1) 13.9MB 31.85%

DONE Q2: Valid Parenthesis leetcode   easy

  • Use a stack to keep the open parenthesis in order.
  • if ([{, then push. if )]}, check the correspondence and if ok, continue.
  • Keep the pairs in a dictionary, that makes it easy to maintain.
 1: class Solution:
 2:     def isValid(self, s: str) -> bool:
 3:         # openers will contain all the opening parenthesis that are
 4:         # not closed yet.
 5:         openers = []
 6:         pairs = {'(':')', '[':']', '{':'}'}
 7:         for par in s:
 8:             if par in '([{':
 9:                 # opening, push to stack
10:                 openers.append(par)
11:             elif openers:
12:                 # closing AND there is at least one opener in the stack.
13:                 # check the corresponding opener
14:                 if pairs[openers.pop()] != par:
15:                     return False
16:             else:
17:                 # closing BUT the stack of openers is empty.
18:                 # This is a disbalance in the force...
19:                 return False
20: 
21:         # at this point nest_stack should be empty, because all parethesis
22:         # should have closed with their matching pairs
23:         if openers:
24:             return False
25:         return True
Runtime O(n) -- 19% Memory O(n) -- 69%

DONE Q1: Two Sum leetcode   easy

  • Use a dictionary to store potential matches for future numbers.
 1: class Solution:
 2:     def twoSum(self, nums: List[int], target: int) -> List[int]:
 3:         if not nums:
 4:             raise Exception("The list 'nums' is empty")
 5:         '''
 6:         dict with pairs (<number previously found>, <index of that number>)
 7:         for every number X in nums[], we will lookup this dictionary first to see
 8:         whether we have seen the value (target-X) before.
 9:         '''
10:         wanted = {}
11:         for j in range(len(nums)):
12:             looking_for = target - nums[j]
13:             if looking_for in wanted:
14:                 return(wanted[looking_for],j)
15:             wanted[nums[j]] = j
16:         raise Exception("No pair found, you lied to me...")
17:         return
Runtime O(n) -- 71% Memory O(n) -- 49%
