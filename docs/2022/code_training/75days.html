DONE 142. Linked List Cycle II (leetcode)   medium

This problem has at least two solutions: with extra \(O(n)\) memory, and without extra memory \(O(1)\).

Solution 1

  • The first one uses a hash-table to store the pointers of the visited nodes, every time you arrive to a new node, you check it in the table. If you find it, then you were there before. You found the cycle.
 1: # Definition for singly-linked list.
 2: # class ListNode:
 3: #     def __init__(self, x):
 4: #         self.val = x
 5: #         self.next = None
 6: 
 7: class Solution:
 8:     def detectCycle(self, head: Optional[ListNode]) -> Optional[ListNode]:
 9:         visited = set(())
10:         if not head:
11:             return None
12:         while head:
13:             if head in visited:
14:                 return head
15:             visited.add(head)
16:             head = head.next
17:         return None
Runtime \(O(n)\) 85ms 39% Memory \(O(n)\) 18MB 11%

Solution 2

  • The other approach is known as "Floyd's algorithm for cycle detection". There is a nice explanation on why it works in this video.
  • It is also known as "the hare and turtle" algorithm. You have two pointers traversing the list. They both start at the beginning. The turtle advances one node at the time, but the hare 2 at the time.
  • If there is no loop, the hare reaches Null. If there is a loop, the hare catches the turtle. Then you move one pointer to the list's head, and make them move one node at the time. They meet at the beginning of the loop.
 1: # Definition for singly-linked list.
 2: # class ListNode:
 3: #     def __init__(self, x):
 4: #         self.val = x
 5: #         self.next = None
 6: 
 7: class Solution:
 8:     def detectCycle(self, head: Optional[ListNode]) -> Optional[ListNode]:
 9:         fast = slow = head
10: 
11:         # let them run!
12:         while fast!=None and fast.next!=None:
13:             slow = slow.next
14:             fast = fast.next.next
15:             if slow == fast:
16:                 break
17: 
18:         # if the list has no loops
19:         if fast == None or fast.next == None:
20:             return None
21: 
22:         # meet each other at the entrance of the loop
23:         p1 = head # one start from the head,
24:         p2 = slow # and the other from the meeting point.
25:         while p1 != p2:
26:             p1 = p1.next
27:             p2 = p2.next
28: 
29:         return p1
Runtime \(O(n)\) 54ms 90% Memory \(O(1)\) 17MB 58%

DONE 876. Middle of the Linked List (leetcode)   easy

  • Make pointers tip and mid run across the linked list. But make mid run at half speed. At the end, mid will be in the center.
 1: # Definition for singly-linked list.
 2: # class ListNode:
 3: #     def __init__(self, val=0, next=None):
 4: #         self.val = val
 5: #         self.next = next
 6: class Solution:
 7:     def middleNode(self, head: Optional[ListNode]) -> Optional[ListNode]:
 8:         tip = mid = head
 9:         count = 0
10:         while tip.next:
11:             tip = tip.next
12:             if count % 2 == 0:
13:                 mid = mid.next
14:             count += 1
15:         return mid
Runtime \(O(n)\) 56ms 25% Memory \(O(1)\) 14MB 96%

DONE 206. Reverse Linked List (leetcode)   easy

  • Point the next node with tmp_nxt
  • Make the current node head point the current tail.
  • Now the current node is the beginning of he tail.
  • Move the pointer head to point to the saved next in tmp_nxt.
  • Repeat.
  • At the end, when there is no head.next, the head needs one more time to point to the current tail, and we are done.
 1: # Definition for singly-linked list.
 2: # class ListNode:
 3: #     def __init__(self, val=0, next=None):
 4: #         self.val = val
 5: #         self.next = next
 6: class Solution:
 7:     def reverseList(self, head: Optional[ListNode]) -> Optional[ListNode]:
 8:         if not head:
 9:             return head
10:         tail = None
11:         while head.next: #to check
12:             tmp_nxt = head.next
13:             head.next = tail
14:             tail = head
15:             head = tmp_nxt
16:         head.next = tail
17:         return head
Runtime \(O(n)\) 43ms 80% Memory \(O(1)\) 15MB 94%

DONE 21. Merge Two Sorted Lists (leetcode)   easy

  • I like more my other approach in Top 100 Liked Questions (Q3), it is faster.
 1: # Definition for singly-linked list.
 2: # class ListNode:
 3: #     def __init__(self, val=0, next=None):
 4: #         self.val = val
 5: #         self.next = next
 6: class Solution:
 7:     def mergeTwoLists(self, list1: Optional[ListNode], \
 8:                       list2: Optional[ListNode]) \
 9:                       -> Optional[ListNode]:
10:         if not list1:
11:             return list2
12:         if not list2:
13:             return list1
14: 
15:         dummy = head = ListNode()
16: 
17:         while list1 and list2:
18:             if list1.val < list2.val:
19:                 head.next = list1
20:                 list1 = list1.next
21:             else:
22:                 head.next = list2
23:                 list2 = list2.next
24:             head = head.next
25: 
26:         if list1:
27:             head.next = list1
28:         elif list2:
29:             head.next = list2
30: 
31:         return dummy.next
Runtime \(O(n)\) 67ms 28% Memory \(O(1)\) 13.9MB 32%

DONE 392. Is Subsequence (leetcode)   easy

  • One from s it is taken one character at the time.
  • It is searched in t starting from the beginning.
  • If it is found, we move to the next character of s.
  • The next search in t starts where it left off.
  • If t is exhausted and there are still elements in s, then s was not a sub-sequence of t.
 1: class Solution:
 2:     def isSubsequence(self, s: str, t: str) -> bool:
 3:         if len(t) < len(s):
 4:             return False
 5:         t_index = 0
 6:         for s_char in s:
 7:             if len(t) <= t_index:
 8:                 return False
 9:             while s_char != t[t_index]:
10:                 t_index += 1
11:                 if len(t) <= t_index:
12:                     return False
13:             t_index += 1
14:         return True
Runtime \(O(n)\) 57ms 37% Memory \(O(1)\) 14MB 45%

DONE 205. Isomorphic Strings (leetcode)   easy

  • It is a bijective function, each character must have one and only one correspondence in the other set.
 1: class Solution:
 2:     def isIsomorphic(self, s: str, t: str) -> bool:
 3:         if len(s) != len(t):
 4:             return False
 5:         # two dictionaries to translate in both directions
 6:         trans_one={}
 7:         trans_two={}
 8:         for sc,tc in zip(s,t):
 9:             # retrive the definitions of both
10:             ab = trans_one[sc] if sc in trans_one else None
11:             ba = trans_two[tc] if tc in trans_two else None
12:             # if both exist, they must be reciprocal
13:             if ab and ba:
14:                 if (ab != tc) or (ba != sc):
15:                     return False
16:                 continue
17:             # if none of them exists, then add the definitions
18:             if not (ab or ba):
19:                 trans_one[sc] = tc
20:                 trans_two[tc] = sc
21:                 continue
22:             # if only one exist, then the characters don't
23:             # correspond to each other
24:             return False
25:         return True
Runtime \(O(n)\) 68ms 44% Memory \(O(1)\) 14.2MB 44%

DONE 724. Find Pivot Index (leetcode)   easy

  • First attempt was to solve through two indices coming from the extremes towards the center. Negative numbers gave me problems, and at the end, realized that this was not the way to solve it, as it is not possible to know what is the value/order of the numbers "towards the center of the list", then, you cannot decide which index should step inwards.
  • By scanning from left to right, the first pivot will be the left-most.
  • If leftsum is never equal to the other part of the list (total-nums[i]-leftsum), then we never found the balance, therefore, return -1.
 1: from typing import List
 2: class Solution:
 3:     def pivotIndex(self, nums: List[int]) -> int:
 4:         total = sum(nums)
 5:         leftsum = 0
 6:         for i in range(len(nums)):
 7:             if leftsum == total - nums[i] - leftsum:
 8:                 return i
 9:             else:
10:                 leftsum += nums[i]
11:         return -1
Runtime \(O(n)\) 159ms 92% Memory \(O(1)\) 15.2MB 93%

DONE 1480. Running Sum of 1d Array (leetcode)   easy

  • No much to say.
 1: from typing import List
 2: class Solution:
 3:     def runningSum(self, nums: List[int]) -> List[int]:
 4:         answer = []
 5:         if not nums:
 6:             raise Exception("nums is null or empty")
 7: 
 8:         answer.append(nums[0])
 9: 
10:         for i in range(1,len(nums)):
11:             answer.append(nums[i]+answer[i-1])
12: 
13:         return answer
Runtime \(O(n)\) 51ms 73% Memory \(O(n)\) 14.2MB 26%
