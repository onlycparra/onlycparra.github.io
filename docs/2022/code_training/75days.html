DONE 409. Longest Palindrome (leetcode)   easy

There are at least two approaches: One with exec \(O(n)\) and mem \(O(n)\). And another with exec: \(O(n\log{n})\) and mem \(O(1)\).

Solution 1

  • This approach uses a set data structure to collect pairs of characters that can be added to the palindrome.
  • If there are also un-paired characters, then put one extra in the center of the palindrome.
 1: class Solution:
 2:     def longestPalindrome(self, s: str) -> int:
 3:         chars = set(())
 4:         pal_lenght = 0
 5: 
 6:         # form pairs of characters to add at each side
 7:         # of the palindrome
 8:         for c in s:
 9:             if c in chars:
10:                 chars.remove(c)
11:                 pal_lenght += 2
12:             else:
13:                 chars.add(c)
14: 
15:         # if there is some un-paired character to put
16:         # in the center
17:         if 0 < len(chars):
18:             pal_lenght += 1
19: 
20:         return pal_lenght
Runtime \(O(n)\) 59ms 29% Memory \(O(n)\) 14MB 22%

Solution 2

  • The second approach could sort the characters in place (but python does not have an in-place sorter for characters in a string. Strings are immutable, and I am to lazy to implement it myself, so for the sake of the example, I'll just sort out-of-place), and then scans the resulting array in a linear fashion to count the pairs of characters.
  • Again, if there is at least one lonely character, then put it in the middle of the palindrome.
 1: class Solution:
 2:     def longestPalindrome(self, s: str) -> int:
 3:         # s.sort() # nope
 4:         s = sorted(s)
 5:         if len(s) == 1:
 6:             return 1
 7:         i = 0
 8:         un_paired = 0
 9:         pal_length = 0
10: 
11:         # compare adjacent characters
12:         while i+1 < len(s):
13:             if s[i] == s[i+1]:
14:                 pal_lenth += 2
15:                 i += 2
16:             else:
17:                 un_paired += 1
18:                 i += 1
19: 
20:         # if the set had an odd number of elements
21:         if i < len(s):
22:             un_paired += 1
23: 
24:         # if any un_paired, put it at the center of
25:         # the palindrome.
26:         if un_paired:
27:             pal_length += 1
28: 
29:         return pal_length
Runtime \(O(n\log{n})\) 51ms 49% Memory \(O(1)\) 14MB 98%

DONE 121. Best Time to Buy and Sell Stock (leetcode)   easy

  • We start with two indices of the first two days of the array prices.
  • Then, as long as the "sale day" is within the prices, then compute the potential profit or loss.
  • If there is some profit, then check if that is greater than our "so-far" best profit, and update accordingly.

  • If there is actually loss, that means that this supposedly selling date is better as a buying day. It is safe to move buy_time forward towards sell_time, because:

    • If there was any profit in the future, it is better to buy now because the price is lower.
    • If there was any better deal in the past, then at some point we must have seen positive profit, and therefore, must have updated max_prof. So the best profit so far is saved.
    • Finally, sell_time moves one day forward, and the cycle repeats.
 1: class Solution:
 2:     def maxProfit(self, prices: List[int]) -> int:
 3:         buy_time = 0
 4:         sell_time = 1
 5:         max_profit = 0
 6: 
 7:         while sell_time < len(prices):
 8:             prof_loss = prices[sell_time] - prices[buy_time]
 9:             if 0 < prof_loss:
10:                 max_profit = max(max_profit, prof_loss)
11:             else:
12:                 buy_time = sell_time
13:             sell_time += 1
14: 
15:         return max_profit
Runtime \(O(n)\) 1270ms 73% Memory \(O(1)\) 25MB 86%

DONE 142. Linked List Cycle II (leetcode)   medium

This problem has at least two solutions: with extra \(O(n)\) memory, and without extra memory \(O(1)\).

Solution 1

  • The first one uses a hash-table to store the pointers of the visited nodes, every time you arrive to a new node, you check it in the table. If you find it, then you were there before. You found the cycle.
 1: # Definition for singly-linked list.
 2: # class ListNode:
 3: #     def __init__(self, x):
 4: #         self.val = x
 5: #         self.next = None
 6: 
 7: class Solution:
 8:     def detectCycle(self, head: Optional[ListNode]) -> Optional[ListNode]:
 9:         visited = set(())
10:         if not head:
11:             return None
12:         while head:
13:             if head in visited:
14:                 return head
15:             visited.add(head)
16:             head = head.next
17:         return None
Runtime \(O(n)\) 85ms 39% Memory \(O(n)\) 18MB 11%

Solution 2

  • The other approach is known as "Floyd's algorithm for cycle detection". There is a nice explanation on why it works in this video.
  • It is also known as "the hare and turtle" algorithm. You have two pointers traversing the list. They both start at the beginning. The turtle advances one node at the time, but the hare 2 at the time.
  • If there is no loop, the hare reaches Null. If there is a loop, the hare catches the turtle. Then you move one pointer to the list's head, and make them move one node at the time. They meet at the beginning of the loop.
 1: # Definition for singly-linked list.
 2: # class ListNode:
 3: #     def __init__(self, x):
 4: #         self.val = x
 5: #         self.next = None
 6: 
 7: class Solution:
 8:     def detectCycle(self, head: Optional[ListNode]) -> Optional[ListNode]:
 9:         fast = slow = head
10: 
11:         # let them run!
12:         while fast!=None and fast.next!=None:
13:             slow = slow.next
14:             fast = fast.next.next
15:             if slow == fast:
16:                 break
17: 
18:         # if the list has no loops
19:         if fast == None or fast.next == None:
20:             return None
21: 
22:         # meet each other at the entrance of the loop
23:         p1 = head # one start from the head,
24:         p2 = slow # and the other from the meeting point.
25:         while p1 != p2:
26:             p1 = p1.next
27:             p2 = p2.next
28: 
29:         return p1
Runtime \(O(n)\) 54ms 90% Memory \(O(1)\) 17MB 58%

DONE 876. Middle of the Linked List (leetcode)   easy

  • Make pointers tip and mid run across the linked list. But make mid run at half speed. At the end, mid will be in the center.
 1: # Definition for singly-linked list.
 2: # class ListNode:
 3: #     def __init__(self, val=0, next=None):
 4: #         self.val = val
 5: #         self.next = next
 6: class Solution:
 7:     def middleNode(self, head: Optional[ListNode]) -> Optional[ListNode]:
 8:         tip = mid = head
 9:         count = 0
10:         while tip.next:
11:             tip = tip.next
12:             if count % 2 == 0:
13:                 mid = mid.next
14:             count += 1
15:         return mid
Runtime \(O(n)\) 56ms 25% Memory \(O(1)\) 14MB 96%

DONE 206. Reverse Linked List (leetcode)   easy

  • Point the next node with tmp_nxt
  • Make the current node head point the current tail.
  • Now the current node is the beginning of he tail.
  • Move the pointer head to point to the saved next in tmp_nxt.
  • Repeat.
  • At the end, when there is no head.next, the head needs one more time to point to the current tail, and we are done.
 1: # Definition for singly-linked list.
 2: # class ListNode:
 3: #     def __init__(self, val=0, next=None):
 4: #         self.val = val
 5: #         self.next = next
 6: class Solution:
 7:     def reverseList(self, head: Optional[ListNode]) -> Optional[ListNode]:
 8:         if not head:
 9:             return head
10:         tail = None
11:         while head.next: #to check
12:             tmp_nxt = head.next
13:             head.next = tail
14:             tail = head
15:             head = tmp_nxt
16:         head.next = tail
17:         return head
Runtime \(O(n)\) 43ms 80% Memory \(O(1)\) 15MB 94%

DONE 21. Merge Two Sorted Lists (leetcode)   easy

  • I like more my other approach in Top 100 Liked Questions (Q3), it is faster.
 1: # Definition for singly-linked list.
 2: # class ListNode:
 3: #     def __init__(self, val=0, next=None):
 4: #         self.val = val
 5: #         self.next = next
 6: class Solution:
 7:     def mergeTwoLists(self, list1: Optional[ListNode], \
 8:                       list2: Optional[ListNode]) \
 9:                       -> Optional[ListNode]:
10:         if not list1:
11:             return list2
12:         if not list2:
13:             return list1
14: 
15:         dummy = head = ListNode()
16: 
17:         while list1 and list2:
18:             if list1.val < list2.val:
19:                 head.next = list1
20:                 list1 = list1.next
21:             else:
22:                 head.next = list2
23:                 list2 = list2.next
24:             head = head.next
25: 
26:         if list1:
27:             head.next = list1
28:         elif list2:
29:             head.next = list2

DONE 392. Is Subsequence (leetcode)   easy

  • One from s it is taken one character at the time.
  • It is searched in t starting from the beginning.
  • If it is found, we move to the next character of s.
  • The next search in t starts where it left off.
  • If t is exhausted and there are still elements in s, then s was not a sub-sequence of t.
 1: class Solution:
 2:     def isSubsequence(self, s: str, t: str) -> bool:
 3:         if len(t) < len(s):
 4:             return False
 5:         t_index = 0
 6:         for s_char in s:
 7:             if len(t) <= t_index:
 8:                 return False
 9:             while s_char != t[t_index]:
10:                 t_index += 1
11:                 if len(t) <= t_index:
12:                     return False
13:             t_index += 1
14:         return True
Runtime \(O(n)\) 57ms 37% Memory \(O(1)\) 14MB 45%

DONE 205. Isomorphic Strings (leetcode)   easy

  • It is a bijective function, each character must have one and only one correspondence in the other set.
 1: class Solution:
 2:     def isIsomorphic(self, s: str, t: str) -> bool:
 3:         if len(s) != len(t):
 4:             return False
 5:         # two dictionaries to translate in both directions
 6:         trans_one={}
 7:         trans_two={}
 8:         for sc,tc in zip(s,t):
 9:             # retrive the definitions of both
10:             ab = trans_one[sc] if sc in trans_one else None
11:             ba = trans_two[tc] if tc in trans_two else None
12:             # if both exist, they must be reciprocal
13:             if ab and ba:
14:                 if (ab != tc) or (ba != sc):
15:                     return False
16:                 continue
17:             # if none of them exists, then add the definitions
18:             if not (ab or ba):
19:                 trans_one[sc] = tc
20:                 trans_two[tc] = sc
21:                 continue
22:             # if only one exist, then the characters don't
23:             # correspond to each other
24:             return False
25:         return True
Runtime \(O(n)\) 68ms 44% Memory \(O(1)\) 14.2MB 44%

DONE 724. Find Pivot Index (leetcode)   easy

  • First attempt was to solve through two indices coming from the extremes towards the center. Negative numbers gave me problems, and at the end, realized that this was not the way to solve it, as it is not possible to know what is the value/order of the numbers "towards the center of the list", then, you cannot decide which index should step inwards.
  • By scanning from left to right, the first pivot will be the left-most.
  • If leftsum is never equal to the other part of the list (total-nums[i]-leftsum), then we never found the balance, therefore, return -1.
 1: from typing import List
 2: class Solution:
 3:     def pivotIndex(self, nums: List[int]) -> int:
 4:         total = sum(nums)
 5:         leftsum = 0
 6:         for i in range(len(nums)):
 7:             if leftsum == total - nums[i] - leftsum:
 8:                 return i
 9:             else:
10:                 leftsum += nums[i]
11:         return -1
Runtime \(O(n)\) 159ms 92% Memory \(O(1)\) 15.2MB 93%

 4:         answer = []
 5:         if not nums:

 7: 
 8:         answer.append(nums[0])
